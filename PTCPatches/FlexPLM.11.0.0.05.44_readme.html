java -jar FlexPLM.11.0.0.05.44.jar <option>

where <option>  can be one of the following:

Option Description
--install Installs the patch. Use this option when installing the patch for the first time.
--forceinstall Forces the installation of the patch. Use this option when installing the patch multiple times.
--help Lists the options for the command.

Installation Instructions

Caution: Do not attempt to manually extract and install the files.

Caution: PTC strongly recommends validating the patch functionality on a test system prior to deploying into production.

Before installing this patch on your production system, it should be installed and tested in an environment that mirrors your production system.

Validate Installation Directory and Prerequisites

  1. Start a windchill shell:
    windchill shell
  2. Enter the following command to determine if your system is at the correct level to install this patch:
    windchill version
  3. Check the output of the command to determine if the Windchill Services 11.0 component is at the Support Datecode F000.
    If this is not the case then this patch should not be installed. Please contact PTC Technical Support for assistance.

Take System Offline

  1. Shut down the Windchill server manager and method server, including any Windchill gateway servlets.
  2. Back up the system and the database.
  3. Delete all cached servlets by executing the following steps:
    1. Stop the Windchill server processes, server manager and method server(s).
    2. Delete the <Windchill>/tomcat/instances directory.
  4. Ensure the default Windchill LDAP Server (Aphelion or Windchill Directory Server) is running.
  5. Make sure that the properties and other files from the Windchill installation directory are not open in a text editor.

Install Patch

The patch JAR can be located in any directory. However, these instructions assume that the FlexPLM.11.0.0.05.44 file has been stored in the <Windchill> directory.

  1. Start a windchill shell.
  2. Install the patch by entering the following command:

    java -jar FlexPLM.11.0.0.05.44.jar --install <Windchill>

    where <Windchill>  refers to the product installation directory where you are installing the patch.

    The installer issues messages as it executes. An information log for the installation is written to the following file:

    <Windchill>/installer/logs/FlexPLM.11.0.0.05.44_PtcInstall.log

    This log contains critical information that you should have available if you need to contact PTC Technical support about this patch.

    If appropriate (based on the content of this patch), the installer automatically compiles RBINFO files, propagates XCONF file changes, and rebuilds the client JAR files.

    If you have previously modified files provided by PTC, this patch may change some of the same files. Consult the patch content list to determine if the content overlaps with your site modifications in the /wtSafeArea/siteMod directory on your system. Please note that InfoEngine tasks and DCA configurations are not included in patch content list and may be overwritten by the patch. If these tasks have been customized using safe area they would need to be redeployed.

  3. To confirm the registration of this patch, execute the following command:

    windchill version

    You should see FlexPLM.11.0.0.05.44 in the list of patches.

  4. Review the SPR Solution description section of this document. Take notes of each patch level that is newer than what is currently installed. Run any new alter(s) unless special instructions indicate to only run an alter under certain circumstances. Ensure new alter(s) gets run in the order of oldest to newest patch release, excluding any previous patches that have already been installed.

    There are two types of updates that may be listed in the SPR Solution section: (Note:  Special steps will be in bold.)
    - MANUAL STEPS (Windchill Services stopped)
    - MANUAL STEPS (Windchill Services running)
  5. Execute all "MANUAL STEPS (Windchill Services stopped)" listed in the SPR Solution section after execution of any required alter(s) in the order of patch release (oldest to newest, excluding any previous patches that have already been installed).
  6. Execute the following command from a Windchill shell to finish the installation process:
    ant -buildfile <Windchill>/src/com/lcs/build/installer/RFAInstaller2.xml postWindchillUpdates

Bring System Online

  1. Start a windchill shell (If not already started by patch deployment process).
  2. Bring the system back to a production state by restarting the web server, servlet engine, and server manager.
  3. Execute all "MANUAL STEPS (Windchill Services running)" listed in the SPR Solution section after execution of any required alter(s) in the order of patch release (oldest to newest, excluding any previous patches that have already been installed).

Installation for a Master-Replica Setup

(Note: Ignore this section if you do not use Master-Replica setup.)

  1. Install the patch on the master system.

    If property autoManageCCS=false, perform the following steps to apply patch on the replica system:
    1. Shut down all Windchill-related applications.
    2. Copy the CcsDsu.zip file from the master directory:

      <Windchill>/codebase/CCSTools/update

      to the same directory on the replica site:

      <Windchill>/codebase/CCSTools/update
    3. On the replica system:
      1. Start a windchill shell.
      2. Run the following:

        <Windchill>/ant/bin/ant -f <Windchill>/bin/CCSTools/install_ccsdsu.xml

        Perform step 1. and 2. below only if the content is not moved automatically from '<Windchill>/codebase/CCSTools/update' to '<Windchill>/codebase/CCSTools/completedUpdate'

        1. Create <Windchill>/codebase/CCSTools/completedUpdate directory, if it does not already exist.
        2. Move the contents of 'update' directory:

          <Windchill>/codebase/CCSTools/update

          to the 'completedUpdate' directory:

          <Windchill>/codebase/CCSTools/completedUpdate

  2. To confirm the registration of this patch on replica system, execute the command below:

    windchill version

    You should see FlexPLM.11.0.0.05.44 in the list of patches.

Installation of Patch on Safe Area-Managed Installations

(Note: Ignore this section if your installation does not use safe area.)

Use installation modes --safeAreaInstall and --completeSafeAreaInstall to install patches on installations having safe areas. These two modes will help patch installations respect the safe area and prevent patches from overwriting customizations. These modes are triggered by the existence of a <Windchill>/wtSafeArea/siteMod directory. If <Windchill>/wtSafeArea directory is renamed to <Windchill>/wtSafeArea.bak, the patch will not respect the safe area (<Windchill>/wtSafeArea) and is installable using the --install install mode. If you have a safe area, we strongly recommend that you do not rename the wtSafeArea directory and that you use the steps below for your patch installation:

  1. Run the patch installation in safe area mode using the --safeAreaInstall flag.
    Safe area installation mode copies files that are common between installable content of the patch and <Windchill>/wtSafeArea/siteMod to <Windchill>/wtSafeArea/ptcCurrent.
  2. Merge any updated or new files in <Windchill>/wtSafeArea/ptcCurrent (updated or added from step 1) with your customizations in <Windchill>/wtSafeArea/siteMod. Once you are sure siteMod looks good with respect to ptcCurrent, move to step 3.
  3. Run the patch installation in complete safe area mode using the --completeSafeAreaInstall flag.
    Complete safe area mode performs normal deployment steps (as done by --install mode) and has an extra step of copying merged files in <Windchill>/wtSafeArea/siteMod (from step 2) to <Windchill>.
